Quantization is the process of compressing an AI model by reducing the precision of its weights (the math parameters) from high-precision floating-point numbers (like 16-bit) to lower-precision numbers (like 4-bit or 8-bit). This shrinks file sizes and lowers memory usage, making it possible to run large models on consumer hardware.

  • The naming convention for compressed AI models (GGUF format) tells you exactly how the model was compressed. Using the example Q4_K_M:
    • Q (Quantization): Simply means the model has been compressed.
    • 4 (4-bit): The model's weights are stored at an average of 4-bit precision instead of 16-bit, which reduces the file size by roughly 70%.
    • K (K-Quants): Refers to a highly optimized method developed by the local AI community that groups weights into blocks. Instead of treating all weights equally, it uses advanced math to keep the numbers more accurate.
    • M (Medium): The size/quality profile of the model. It uses a mixed-precision strategy. Less important data gets 4-bit compression, while "critical layers" (like attention mechanisms) retain 5- or 6-bit precision to preserve accuracy.
  • The Quantization Trade-Off
    • Q2 - Q3: Extremely small, fits on edge devices, but has a major drop in reasoning accuracy.
    • Q4_K_M: The widely accepted "sweet spot". It balances 97-99% of unquantized output quality while fitting comfortably on standard hardware (like an 8GB GPU or a Mac).
    • Q5 - Q6: Retains higher accuracy for precise reasoning, logic, or coding but requires more memory and bandwidth.
    • Q8: Near full precision; useful if you are fine-tuning or require absolute fidelity, though it is very heavy on RAM/VRAM.

Capabilities

Ollama is a tool that allows you to run large language models locally on your own hardware. Its capabilities depend entirely on the specific model you choose to download from the Ollama Library, spanning a wide variety of open-source models.

  • Because Ollama supports a modular ecosystem of models (like Llama, Qwen, Gemma, and DeepSeek), its capabilities can include:
    • Multimodal Processing: Vision models can accept both text and images to describe visual content, diagrams, or documents.
    • Agentic Capabilities & Tool Use: Advanced models support function calling (triggering APIs or Python code) and structured JSON outputs.
    • Built-in Web Search: Optional REST API tools allow models to access real-time internet information to reduce hallucinations.
    • Full Chain-of-Thought ("Thinking"): Many of the latest reasoning models natively emit a distinct "thinking" field, allowing you to audit their logic steps or hide the reasoning trace.
    • Coding Assistance: Specialized coding models offer auto-completion and real-time suggestions natively integrated with IDEs like Visual Studio Code.
    • Privacy: Because everything runs on your local machine, you do not have to send sensitive data to third-party cloud servers.

Parameters

Ollama uses specific parameters to control how models generate text, handle memory, and balance creativity. These parameters can be customized using the /set parameter command, defined in a Modelfile, or passed via the API.

  • Core Generation Parameters
    These parameters adjust the creativity, style, and predictability of responses:
    • temperature: Controls response creativity. Lower values (e.g., 0.1) yield more precise, deterministic, and factual answers (best for coding). Higher values (e.g., 0.8 to 1.0) make the model more adventurous and creative (best for brainstorming). Default: 0.8.
    • top_k: Limits the pool of potential words the model considers for the next token to the most likely N options. Default: 40.
    • top_p: Works alongside top_k by considering only the smallest set of words whose cumulative probability reaches this value. Default: 0.9.
    • min_p: An alternative to top_p. Sets the minimum probability for a token to be considered, relative to the most likely token. Default: 0.0.
    • repeat_penalty: Determines how strongly the model is penalized for repeating the same words or phrases. A higher value (e.g., 1.5) is strict, while lower values (e.g., 0.9) are lenient. Default: 1.1.
    • repeat_last_n: Controls how far back into the generation history the model looks to prevent repetition. Default: 64.
  • Memory & Context Parameters
    These parameters manage the context window (memory size) and inference length:
    • num_ctx: Sets the size of the context window (in tokens) the model uses for generating the next token. Default is typically 2048 or 4096, depending on your configuration.
    • num_predict: The maximum number of tokens the model should generate per prompt. Default: -1 (generate until completion).
    • seed: Sets the random number seed. Setting this to a specific number makes generation reproducible and guarantees the same output for the same prompt. Default: 0.
    • stop: Defines custom sequences or symbols that will force the model to stop generating text.
  • Setting Parameters in Ollama
    You can adjust these settings at the CLI, via an API, or within a custom blueprint:
    • In an active chat session: Use the /set parameter command (e.g., /set parameter temperature 0.3).
    • In a Modelfile: Add lines like PARAMETER temperature 0.3 to create a reusable custom model.
    • Via API: Pass them in an options object in your POST request to endpoints like /api/generate
