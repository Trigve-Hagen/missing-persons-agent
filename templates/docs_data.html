Codebase Knowledge

For the first release I only want to save data to the databases related to the missing person and persons of interest to have the agent search for clues based upon that data only.

You said: What would I need to create so the model understands everything the codebase can do to make suggestions on how to optimize or adjust it to help find missing people?

To make an AI model fully understand your codebase's capabilities and logic for this critical mission, you need to build a Comprehensive Context Layer. AI models lack the "spatial awareness" of a developer, so you must explicitly feed them the high-level architecture alongside the code. Here is exactly what you need to create and organize:

  1. The Global Architecture Map (README.md or ARCHITECTURE.md)
    Create a single markdown file that acts as the "brain" or entry point for the AI. It must outline:
    • The Core Flow: How data enters the system (e.g., public tips, police feeds, scraping) and how it outputs (e.g., SMS alerts, map pins).
    • Page/Module Index: A literal directory listing what every single page, dashboard, or endpoint does.
    • Tech Stack & Integrations: List all third-party APIs used (e.g., Twilio for SMS, Google Maps API, facial recognition services).
  2. Data Schema & Relationships (SCHEMA.md)
    Optimizing search and matching requires the AI to know exactly how data is structured. Provide:
    • Database Models: Text representations of your database tables (e.g., Person, Sighting, User_Report, Location_History).
    • Field Definitions: Explain critical fields, especially formats for timestamps, geolocation coordinates, and status flags (e.g., Status: Active, Resolved).
  3. The Routing & API Manifest
    AI models struggle to guess how pages connect. Give it a map of your traffic:
    • Frontend Routes: A file showing your URL paths mapped to components (e.g., /search -> SearchPanel.tsx).
    • Backend Endpoints: A list of API routes (e.g., POST /api/v1/sightings) with expected payloads and responses.
  4. Automated Context Bundling (The AI Tooling)
    Do not copy and paste code manually. Use tools to package your codebase into a format AI models love:
    • Repository Packers: Use open-source tools like repomix or code2prompt. These tools compress your entire codebase, folder structure, and markdown files into a single, highly structured text file.
    • Vector Embeddings (For Massive Codebases): If your codebase is too large for an AI's memory window, use a tool like Greptile or Sweep to index your codebase so the AI can search for relevant code snippets on demand.
  5. Optimization & Search Prompt Template
    Once the structure is ready, you must guide the AI's focus toward the specific goal of finding missing people. Use this prompt template to get recommendations: "You are an expert software architect and search-algorithm engineer. Review this codebase context. Our goal is to maximize the speed, accuracy, and reach of our missing persons platform. Analyze our code and suggest optimizations for:
    Data Latency: How can we ingest and display sighting leads faster?
    Search Relevance: Are our database queries or search algorithms missing partial matches (e.g., misspelled names, close geographic proximity)?
    Reliability: Where are the single points of failure in our notification or ingestion pipelines?"
